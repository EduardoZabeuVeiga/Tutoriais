Configuração de Servidor Automático (NSSM)

Guia Mestre para o servidor de 7 Days to Die no kamikaiserbook.

1. Instalação Limpa (Disco C:)

Para evitar problemas de permissão ou exclusão acidental, instalaremos na raiz do sistema.

  1. Baixe a versão prerelease build 2.24-101 em nssm.cc (Essencial para Win10/11).
  2. Crie uma pasta chamada C:\nssm.
  3. Abra o ZIP baixado, entre na pasta win64.
  4. Copie apenas o arquivo nssm.exe para dentro de C:\nssm\.
Caminho final deve ser: C:\nssm\nssm.exe

2. Execução do Instalador

O NSSM precisa ser chamado via linha de comando com privilégios.

  1. Abra o Menu Iniciar, digite CMD.
  2. Clique com o botão direito e escolha Executar como Administrador.
  3. Cole o comando abaixo:
C:\nssm\nssm.exe install "7DaysServer"

3. Configuração Detalhada (As Abas)

Aba "Application" (Os 3 Campos)

1. Path (Caminho): Selecione o arquivo startdedicated.bat. É o script que inicia tudo.
2. Startup directory (Diretório): A pasta raiz do servidor (ex: ...\common\7 Days to Die Dedicated Server). O NSSM preenche sozinho, mas confira. Se estiver errado, o servidor não acha as configs.
3. Arguments (Argumentos): DEIXE EM BRANCO. Como você usa um arquivo .bat, os argumentos já estão escritos dentro dele.

Outras Abas Essenciais

  • Aba Details: Em Startup type, selecione Automatic.
  • Aba Dependencies: Digite LanmanWorkstation na caixa de texto.
    (Isso obriga o servidor a esperar a Internet conectar antes de iniciar).
  • Aba Log on (CRÍTICO PARA O SAVE): Marque "This account".
    Digite seu usuário (Kamikaiser) e a senha do Windows.
    (Isso garante que o jogo encontre seus saves na pasta AppData do seu usuário).
Clique em "Install service" para finalizar

4. Diagnóstico e Saves

⚠️ O MUNDO SUMIU? (Plano B)
Se mesmo configurando o login o save não aparecer, o servidor pode estar olhando para a pasta errada.
  1. Pare o servidor no Gerenciador de Tarefas.
  2. Abra o arquivo serverconfig.xml na pasta do jogo.
  3. Procure a linha UserDataFolder.
  4. Mude para o caminho fixo: C:\Users\Kamikaiser\AppData\Roaming\7DaysToDie
💡 DICA VISUAL (Web Dashboard)
Como o servidor roda "invisível" (sem janela), ative o Web Dashboard no serverconfig.xml mudando WebDashboardEnabled para true. Assim você vê o status pelo navegador.

5. Comandos de Controle

Use estes comandos no CMD (Admin) para gerenciar seu servidor:

Iniciar manualmente:
C:\nssm\nssm.exe start "7DaysServer"
Parar o servidor:
C:\nssm\nssm.exe stop "7DaysServer"
Editar configurações (Se mudar algo):
C:\nssm\nssm.exe edit "7DaysServer"
REMOVER TUDO (Desinstalar):
C:\nssm\nssm.exe remove "7DaysServer" confirm

6. Como saber se está rodando?

Como não há janela preta, verifique assim:

  1. Abra o Gerenciador de Tarefas (Ctrl + Shift + Esc).
  2. Vá na aba Detalhes.
  3. Procure por 7DaysToDieServer.exe.
  4. Se estiver consumindo memória (subindo acima de 2GB), ele está carregando o mapa.
