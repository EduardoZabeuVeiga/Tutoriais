Tutorial: Migrando Projeto Web para Electron

Como transformar sua aplicação web em um app desktop

Voltar ao Índice Tutorial Base Electron
📋 Pré-requisitos: Este tutorial assume que você já seguiu o Tutorial Completo: Criando um App Electron e tem a estrutura básica configurada.

1. Visão Geral da Migração

1.1 O que vamos migrar?

Vamos ensinar como pegar qualquer projeto web existente (HTML, CSS, JS) e transformá-lo em uma aplicação Electron desktop. O processo envolve:

  • HTML: Copiar e adaptar para o index.html do Electron
  • CSS: Migrar estilos para styles.css ou criar arquivos separados
  • JavaScript: Adaptar para renderer.js e configurar segurança
  • Assets: Migrar imagens, fontes e outros recursos

3. Migrando o HTML

3.1 Adaptando o arquivo principal

📂 DE ONDE PEGAR: Do seu arquivo index.html original

📁 PARA ONDE COLOCAR: No arquivo index.html do Electron (substituir o conteúdo)

⚠️ Importante: Você deve adaptar os caminhos dos links CSS e JS para funcionarem no Electron.

PASSO A PASSO SIMPLIFICADO:

  1. Abra seu arquivo HTML original
  2. Copie TUDO que está dentro do <body> (sem as tags <body>)
  3. No Electron, substitua <body> por:

🔹 Tag de abertura:

<div class="container">

📝 Depois cole todo seu conteúdo HTML aqui

🔹 Tag de fechamento:

</div>
✅ Resultado: Em vez de <body>conteúdo</body> você terá <div class="container">conteúdo</div>

ANTES (HTML original):

<link rel="stylesheet" href="style.css">
<script src="script.js"></script>

DEPOIS (Electron):

<link rel="stylesheet" href="styles.css">
<script src="renderer.js"></script>
📍 Onde colocar seu conteúdo no index.html do Electron

O index.html do tutorial Electron tem esta estrutura:

<body>
    <div class="container">
        <header>
            <h1>🚀 Meu Primeiro App Electron</h1>
            <p>Bem-vindo ao seu aplicativo desktop!</p>
        </header>
        
        <main>
            <!-- SEÇÕES EXISTENTES: contador, notificações, etc. -->
            
            <!-- 👇 COLE SEU CONTEÚDO AQUI 👇 -->
            <section>
                <!-- Seu HTML aqui -->
            </section>
            <!-- 👆 COLE SEU CONTEÚDO AQUI 👆 -->
            
        </main>
        
        <footer>
            <p>Criado com ❤️ usando Electron</p>
        </footer>
    </div>
    
    <script src="renderer.js"></script>
</body>
3.2 Migrando páginas adicionais

📂 DE ONDE PEGAR: Arquivos como sobre.html, contato.html, etc.

📁 PARA ONDE COLOCAR: Crie uma pasta pages no projeto Electron

mkdir pages

E copie os arquivos HTML adicionais para esta pasta, ajustando os caminhos relativos.

4. Migrando o CSS

4.1 CSS Principal

📂 DE ONDE PEGAR: Do seu arquivo style.css (ou similar) original

📁 PARA ONDE COLOCAR: No arquivo styles.css do Electron

OPÇÕES DE MIGRAÇÃO:

C Opção 1: Substituição Total

Substitua completamente o conteúdo do styles.css do Electron pelo seu CSS original.

  1. Abra seu style.css original
  2. Selecione tudo (Ctrl+A)
  3. Copie (Ctrl+C)
  4. Abra styles.css do Electron
  5. Substitua tudo (Ctrl+A + Ctrl+V)

C Opção 2: Mesclar Estilos

Adicione seus estilos ao final do styles.css existente do Electron.

  1. Abra styles.css do Electron
  2. Vá até o final do arquivo
  3. Adicione o comentário: /* ESTILOS MIGRADOS */
  4. Cole seu CSS original abaixo
4.2 CSS Adicional e Frameworks

Se você usa múltiplos arquivos CSS ou frameworks:


<link rel="stylesheet" href="styles.css">
<link rel="stylesheet" href="assets/css/framework.css">
<link rel="stylesheet" href="assets/css/componentes.css">


<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css" rel="stylesheet">

5. Migrando o JavaScript

5.1 JavaScript Principal

📂 DE ONDE PEGAR: Do seu arquivo script.js (ou similar) original

📁 PARA ONDE COLOCAR: No arquivo renderer.js do Electron

⚠️ Importante: No Electron, há separação entre "processo principal" (main.js) e "processo renderer" (renderer.js). Seu JavaScript de interface vai no renderer.js.

ESTRATÉGIAS DE MIGRAÇÃO:

  • ESTRATÉGIA 1: Substituição Total
    Substitua todo o conteúdo do renderer.js pelo seu script.js original
  • ESTRATÉGIA 2: Mesclar Código
    Mantenha as funcionalidades do Electron e adicione as suas ao final do arquivo
  • ESTRATÉGIA 3: Modular
    Crie arquivos separados e importe no index.html usando múltiplas tags <script>
5.2 Exemplo Prático de Migração JS

Vamos supor que você tem este código no seu script.js original:

// script.js original
document.addEventListener('DOMContentLoaded', function() {
    const botao = document.getElementById('meuBotao');
    const resultado = document.getElementById('resultado');
    
    botao.addEventListener('click', function() {
        resultado.textContent = 'Botão clicado!';
    });
    
    // Fetch para API externa
    fetch('https://api.exemplo.com/dados')
        .then(response => response.json())
        .then(data => console.log(data));
});

📁 COLE NO renderer.js do Electron:

// renderer.js do Electron - ADICIONE ao final do arquivo existente

// ========== CÓDIGO MIGRADO ==========
document.addEventListener('DOMContentLoaded', function() {
    // Suas funcionalidades originais
    const botao = document.getElementById('meuBotao');
    const resultado = document.getElementById('resultado');
    
    if (botao && resultado) {
        botao.addEventListener('click', function() {
            resultado.textContent = 'Botão clicado!';
            showFeedback('Ação executada!', 'success'); // Usa função do Electron
        });
    }
    
    // APIs externas funcionam normalmente no Electron
    fetch('https://api.exemplo.com/dados')
        .then(response => response.json())
        .then(data => {
            console.log(data);
            showFeedback('Dados carregados!', 'success');
        })
        .catch(error => {
            console.error('Erro:', error);
            showFeedback('Erro ao carregar dados', 'error');
        });
});

6. Migrando Assets (Imagens, Fontes, etc.)

6.1 Estrutura de Assets

📂 DE ONDE PEGAR: Pastas como images/, fonts/, icons/ do projeto original

📁 PARA ONDE COLOCAR: Crie uma pasta assets no projeto Electron

mkdir assets
mkdir assets/images
mkdir assets/fonts  
mkdir assets/icons
mkdir assets/css
mkdir assets/js
6.2 Ajustando Caminhos

Após copiar os assets, você precisa ajustar os caminhos no seu HTML e CSS:

🔴 ANTES (Projeto Original)


<img src="images/logo.png">

/* CSS */
background-image: url('images/banner.jpg');

/* JavaScript */
document.querySelector('img').src = 'images/foto.png';

🟢 DEPOIS (Electron)


<img src="assets/images/logo.png">

/* CSS */
background-image: url('assets/images/banner.jpg');

/* JavaScript */
document.querySelector('img').src = 'assets/images/foto.png';

7. Configurações Especiais do Electron

7.1 Atualizando package.json

Se seu projeto web usa dependências npm, adicione-as ao package.json do Electron. Inclua também o bloco "build" para configurar o instalador Windows:

{
  "name": "meu-app-migrado",
  "version": "1.0.0",
  "main": "main.js",
  "scripts": {
    "start": "electron .",
    "dist": "electron-builder"
  },
  "build": {
    "appId": "com.empresa.meu-app",
    "productName": "Meu App Migrado",
    "files": [
      "**/*",
      "!node_modules/electron-builder/**",
      "!*.md"
    ],
    "win": {
      "target": "nsis",
      "icon": "build/icon.ico"
    },
    "nsis": {
      "oneClick": false,
      "allowToChangeInstallationDirectory": true,
      "createDesktopShortcut": true,
      "createStartMenuShortcut": true
    }
  },
  "dependencies": {
    "axios": "^1.6.0",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "electron": "^31.0.0",
    "electron-builder": "^24.13.3"
  }
}

Com "oneClick": false e "allowToChangeInstallationDirectory": true, o instalador gerado exibe um assistente passo a passo que permite ao usuário escolher onde o app será instalado. Para gerar o instalador execute:

npm install
npm run dist

O arquivo .exe de instalação será gerado na pasta dist/.

7.2 Adicionando Ícone ao App

O ícone aparece na barra de título da janela, na taskbar do Windows e no instalador. Siga os passos:

📂 ESTRUTURA NECESSÁRIA: Crie a pasta build/ na raiz do projeto e coloque o arquivo icon.ico dentro dela. O arquivo deve ter pelo menos os tamanhos 16×16, 32×32 e 256×256 (use o próprio app para gerar o .ico). 
meu-app/
├── build/
│   └── icon.ico   ← ícone do instalador e do app
├── main.js
├── index.html
└── package.json

No main.js , passe o caminho do ícone ao criar a janela:

const path = require('path');

const win = new BrowserWindow({
  width: 1280,
  height: 800,
  icon: path.join(__dirname, 'build', 'icon.ico'),  // ← ícone da janela e taskbar
  webPreferences: {
    nodeIntegration: false,
    contextIsolation: true
  }
});

O package.json já referencia o mesmo arquivo para o instalador via "icon": "build/icon.ico" no bloco "win". Com isso, o mesmo .ico é usado em todos os lugares:

✅ Onde o ícone aparece:
  • ✅ Barra de título da janela
  • ✅ Taskbar do Windows (enquanto o app está aberto)
  • ✅ Atalho na área de trabalho
  • ✅ Atalho no menu Iniciar
  • ✅ Janela do instalador (.exe)
7.3 Configurando Segurança (webPreferences)

Se seu projeto usa recursos avançados, ajuste o main.js :

// No main.js, configure webPreferences conforme necessário
const win = new BrowserWindow({
  width: 1280,
  height: 800,
  webPreferences: {
    nodeIntegration: false,          // Segurança
    contextIsolation: true,          // Segurança  
    enableRemoteModule: false,       // Segurança
    webSecurity: true,              // Para CORS se necessário
    allowRunningInsecureContent: false
  }
});

9. Exemplo Completo de Migração

9.1 Projeto Web Simples → Electron

Vamos ver um exemplo prático completo:

📂 PROJETO ORIGINAL: Um site simples com calculadora 

<!DOCTYPE html>
<html>
<head>
    <title>Calculadora Web</title>
    <link rel="stylesheet" href="calc.css">
</head>
<body>
    <div class="calculator">
        <input type="text" id="display" readonly>
        <button onclick="clearDisplay()">C</button>
        <button onclick="appendToDisplay('7')">7</button>
        <!-- mais botões... -->
    </div>
    <script src="calc.js"></script>
</body>
</html>
📁 MIGRAÇÃO PARA ELECTRON: 

<!DOCTYPE html>
<html lang="pt-BR">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Calculadora Electron</title>
    <link rel="stylesheet" href="styles.css">
</head>
<body>
    <div class="container">
        <h1>🧮 Calculadora Desktop</h1>
        
        <!-- CONTEÚDO MIGRADO DO PROJETO ORIGINAL -->
        <div class="calculator">
            <input type="text" id="display" readonly>
            <button onclick="clearDisplay()">C</button>
            <button onclick="appendToDisplay('7')">7</button>
            <!-- mais botões... -->
        </div>
    </div>
    
    <script src="renderer.js"></script>
</body>
</html>

📄 No renderer.js, adicione as funções da calculadora:

// renderer.js - Adicione ao final do arquivo

// ========== FUNÇÕES MIGRADAS DA CALCULADORA ==========
function clearDisplay() {
    document.getElementById('display').value = '';
    showFeedback('Display limpo', 'info');
}

function appendToDisplay(value) {
    document.getElementById('display').value += value;
}

function calculate() {
    try {
        const result = eval(document.getElementById('display').value);
        document.getElementById('display').value = result;
        showFeedback('Cálculo realizado!', 'success');
    } catch (error) {
        document.getElementById('display').value = 'Erro';
        showFeedback('Erro no cálculo', 'error');
    }
}

10. Problemas Comuns e Soluções

10.1 Erros Frequentes
❌ PROBLEMA: "Cannot read property of undefined"
✅ SOLUÇÃO: Verifique se os IDs dos elementos HTML estão corretos após a migração.
❌ PROBLEMA: CSS não carrega
✅ SOLUÇÃO: Verifique os caminhos relativos dos arquivos CSS no HTML.
❌ PROBLEMA: Imagens não aparecem
✅ SOLUÇÃO: Ajuste os caminhos para assets/images/ em todo o projeto.

11. Conclusão

🎉 Parabéns! Você aprendeu como migrar qualquer projeto web para Electron. O processo sempre segue o mesmo padrão:
  • HTML → index.html (adaptar caminhos)
  • CSS → styles.css (mesclar ou substituir)
  • JS → renderer.js (adicionar ao final)
  • Assets → assets/ (ajustar caminhos)
