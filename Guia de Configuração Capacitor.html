Guia de Configuração Capacitor

Pré-requisitos Obrigatórios

Certifique-se de que os seguintes softwares estão instalados no sistema:

  • Node.js: Necessário para execução de pacotes npm e npx.
    Download Node.js
  • Android Studio: Necessário para compilação e gerenciamento do SDK.
    Download Android Studio

Verificação do Ambiente Android SDK

1. Valide se a variável ANDROID_HOME está acessível na sessão atual do PowerShell:

Get-ChildItem -Path Env:ANDROID_HOME

2. Caso o retorno seja ObjectNotFound, execute a atribuição do caminho do SDK:

$env:ANDROID_HOME = "C:\Users\Eduar\AppData\Local\Android\Sdk"

3. Confirme a persistência da variável na sessão antes de prosseguir:

Get-ChildItem -Path Env:ANDROID_HOME

1. Instalar o Capacitor no seu projeto

Abra o terminal do seu computador (Prompt de Comando, PowerShell ou terminal do VS Code), navegue até a pasta onde estão os arquivos do seu site (HTML/CSS/JS) e execute:

npm install @capacitor/core @capacitor/cli

2. Inicializar as configurações do Capacitor

No mesmo terminal e na mesma pasta, execute o comando abaixo para criar o arquivo de configuração da estrutura do aplicativo:

npx cap init

3. Preencher os parâmetros de inicialização

O terminal fará algumas perguntas. Responda conforme o padrão abaixo:

? Name: Controle Sineedy
? Package ID: com.kamikaiser.controle_sineedy
? Web asset directory: www
? Create free Ionic account? n

4. Estruturação do Diretório Web

Crie uma nova pasta chamada www na raiz do seu projeto. Mova o arquivo index.html (OBRIGATÓRIO USO DO NOME INDEX.HTML) e todos os outros arquivos do seu site (CSS, JS, imagens) para dentro desta pasta www.

5. Instalar os pacotes da plataforma Android

Ainda no terminal, execute este comando para adicionar o motor do Android ao seu projeto e criar a pasta nativa:

npm install @capacitor/android && npx cap add android

Observação técnica (PowerShell): Caso ocorra um erro de sintaxe indicando que o token && não é válido, execute as instruções separadamente:

npm install @capacitor/android
npx cap add android

6. Personalizar o ícone e a Splash Screen (Tela de Abertura)

Para substituir o ícone padrão do Capacitor pela sua própria arte, siga estas etapas:

A) Prepare as imagens:

  • Crie uma pasta na raiz do seu projeto (ao lado da pasta www) com o nome: assets
  • Salve seu ícone (mínimo 1024x1024px) com o nome: icon.png
  • Salve sua tela de abertura (mínimo 2732x2732px) com o nome: splash.png

B) Instale a ferramenta de geração:

npm install @capacitor/assets --save-dev

C) Gere as imagens:

npx @capacitor/assets generate --android

D) Sincronize com o projeto:

npx cap sync android

7. Configurar a Duração da Splash Screen

Por padrão a tela de abertura some muito rápido. Para controlar o tempo de exibição, siga os passos abaixo:

A) Instale o plugin de Splash Screen:

npm install @capacitor/splash-screen

B) Edite o arquivo capacitor.config.json na raiz do projeto e substitua todo o conteúdo pelo bloco abaixo. O valor launchShowDuration está em milissegundos — altere conforme desejar (ex: 3000 = 3 segundos):

{
  "appId": "com.kamikaiser.controle_sineedy",
  "appName": "Controle Sineedy",
  "webDir": "www",
  "plugins": {
    "SplashScreen": {
      "launchShowDuration": 3000,
      "launchAutoHide": true,
      "androidScaleType": "centerCrop",
      "androidSplashResourceName": "splash",
      "splashImmersive": true,
      "backgroundColor": "#000000",
      "androidNavigationBarColor": "#000000"
    }
  }
}

C) Sincronize a configuração com o Android:

npx cap sync android

D) Corrigir faixa branca da barra de navegação do Android:

O plugin do Capacitor não controla a cor da barra de navegação (botões Home/Voltar/Recentes). É necessário editar o arquivo nativo:
android\app\src\main\res\values\styles.xml

Localize o bloco AppTheme.NoActionBarLaunch e deixe assim:

<style name="AppTheme.NoActionBarLaunch" parent="Theme.SplashScreen">
    <item name="android:background">@drawable/splash</item>
    <item name="android:navigationBarColor">#000000</item>
    <item name="android:windowDrawsSystemBarBackgrounds">true</item>
</style>

Substitua #000000 pela cor de fundo da sua imagem de splash. Após editar este arquivo, gere o APK direto sem precisar rodar npx cap sync.

8. Sincronizar Código e Abrir Android Studio

Execute os comandos abaixo em sequência. O primeiro sincroniza as alterações do seu site (HTML/CSS/JS) com o projeto Android, e o segundo abre a IDE para gerar o APK:

npx cap sync android
npx cap open android

9. Gerar o APK pelo terminal (Sem Android Studio)

Caso prefira não usar a interface visual do Android Studio, você pode compilar o instalador (.apk) diretamente pelo terminal do VS Code:

⚠️ Pré-requisito obrigatório: Configurar JAVA_HOME

Faça isso antes de rodar qualquer comando abaixo. O Windows precisa saber onde está o Java para o Gradle funcionar:

  1. Abra o menu Iniciar, pesquise por Variáveis de Ambiente e clique em "Editar as variáveis de ambiente do sistema".
  2. Na tela "Propriedades do Sistema", clique em Variáveis de Ambiente...
  3. Em "Variáveis do Sistema", clique em Novo.
  4. Nome: JAVA_HOME
  5. Valor: C:\Program Files\Android\Android Studio\jbr
  6. Reinicie o VS Code após salvar tudo.

A) Acesse a pasta nativa do Android:

cd android

B) Execute o comando de compilação (Build):

.\gradlew assembleDebug

Local do arquivo: Após o término, seu APK estará em:
android\app\build\outputs\apk\debug\app-debug.apk

C) Volte para a pasta principal do projeto:

cd ..

10. Onde encontrar o arquivo APK para instalação

Após o terminal exibir a mensagem de sucesso no Build, o arquivo instalador estará guardado em uma pasta profunda dentro do seu projeto:

Caminho completo:
C:\SEUPROJETO\android\app\build\outputs\apk\debug\app-debug.apk

Como chegar lá pelo Windows:

  1. Abra a pasta do seu projeto (controle de gastos).
  2. Entre em: android > app > build > outputs > apk > debug.
  3. O arquivo app-debug.apk é o seu aplicativo pronto.

Dica: Você pode copiar este arquivo para o seu celular via cabo USB, Google Drive ou WhatsApp Web e instalá-lo diretamente no aparelho para testar.

🔄 ATUALIZANDO MUDANÇAS NO CÓDIGO

⚡ RESUMO RÁPIDO - Execute esses 3 comandos em ordem:

Sempre que você alterar:

  • ✅ Ícone ou Splash Screen
  • ✅ Arquivos HTML, CSS ou JavaScript
  • ✅ Arquivo capacitor.config.json
  • ✅ Qualquer outro arquivo do projeto

Execute NESTA ORDEM:

1️⃣ ATUALIZAR ÍCONE E SPLASH:

npx capacitor-assets generate --android

2️⃣ SINCRONIZAR HTML, CSS, JS E CONFIGURAÇÕES:

npx cap sync android

3️⃣ GERAR O APK (execute em sequência):

cd android
.\gradlew assembleDebug
cd ..

💾 Arquivo gerado: android\app\build\outputs\apk\debug\app-debug.apk

💡 DICA IMPORTANTE:

Se você só alterou HTML/CSS/JS ou capacitor.config.json, você pode pular o comando 1️⃣ e ir direto para 2️⃣ e 3️⃣.

Mas se também alterou ícone ou splash, SEMPRE execute os 3 em ordem!

⚠️ SE DER ERRO:

Certifique-se de que:
✓ JAVA_HOME está configurado (veja passo 9)
✓ ANDROID_HOME está disponível no PowerShell
✓ Você está na pasta do projeto ao executar os comandos
✓ Node.js está instalado (teste: node -v)
