Roteiro Definitivo: Servidor 7 Days to Die

📋 Informações Base da Rede

  • Máquina Host: Computador onde o servidor roda.
  • IP Fixo Interno: IP da máquina host na rede local.
  • Portas Necessárias:
    26900, 26901, 26902, 26903

🕵️ Fase 1: O Ponto de Partida (Teste Prévio)

Antes de configurar o roteador, verifique o estado físico da porta para garantir que nenhum outro serviço a esteja utilizando.

  1. Acesse o site de verificação de portas:
    2. Abrir Port Checker
  2. Teste a seguinte porta:
    3. 26900
  3. Resultado esperado: Deve indicar Closed (Fechada).

⚙️ Fase 2: Configuração do Roteador

Acesse o painel do seu roteador e vá em Redirecionamento de Portas. Crie 4 regras individuais:

  • Protocolo: TCP / UDP (Ambos)
  • Porta Externa / Interna: Criar regras para 26900, 26901, 26902 e 26903.
  • IP Externo: Deixar VAZIO.
  • IP Interno: O IP Fixo da sua máquina host.
Importante: Após salvar as regras, desligue o roteador da tomada por 10 segundos para limpar a memória cache.

Volte ao PortCheckTool.com e teste a porta 26900 novamente. O resultado agora deve ser Open (Aberta).

🛡️ Fase 3: Liberação no Windows Firewall

Abra o PowerShell como Administrador na máquina host e execute o comando abaixo para liberar o tráfego UDP em bloco:

New-NetFirewallRule -DisplayName "7Days_UDP" -Direction Inbound -LocalPort 26900-26903 -Protocol UDP -Action Allow

📝 Fase 4: Arquivo de Configuração (serverconfig.xml)

Abra o arquivo de configuração do servidor e edite as seguintes propriedades:

  • Porta:
    <property name="ServerPort" value="26900"/>
  • Senha:
    <property name="ServerPassword" value="SUA_SENHA_AQUI"/>
  • IP do Servidor: A propriedade deve ficar totalmente vazia para evitar bloqueios:
    <property name="ServerIP" value=""/>

🚀 Fase 5: Inicialização e Testes Finais

Ligue o servidor do jogo. Siga a ordem abaixo para diagnosticar a conexão:

  1. Teste 1: O Windows abriu a porta? (No Prompt de Comando - CMD):
    netstat -ano | findstr :26900
    Resultado esperado: Uma linha mostrando LISTENING.
  2. Teste 2: Diagnóstico de CGNAT (Operadora): Verifique os Detalhes da Internet no roteador. O Endereço IPv4 deve ser idêntico ao seu IP Público. Se começar com 100.x.x.x, há bloqueio de CGNAT.
  3. Teste 3: O Teste de Visibilidade do Serviço: Abrir CanYouSeeMe.org Teste a porta 26900. Resultado esperado: Success.
  4. Teste 4: A Prova de Fogo (Conexão via Celular):
    • Desconecte o cabo de rede do PC de jogo.
    • Conecte o PC roteando a internet móvel 4G/5G do seu celular.
    • No jogo, vá em Join Game > Connect to IP.
    • Digite o seu IP Público e a porta 26900.
    Resultado esperado: O jogo pede a senha e conecta.
