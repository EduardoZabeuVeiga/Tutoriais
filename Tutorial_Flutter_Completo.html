Requisitos do Sistema:

  • Git para Windows
  • PowerShell 5.0 ou superior

⚙️ 2. Instalação do Flutter

Passo 1: Extrair o Flutter SDK

Extraia o arquivo baixado para:

C:\flutter
⚠️ Importante: Não instale o Flutter em um diretório que requeira privilégios elevados, como C:\Program Files\.

Passo 2: Verificar a Estrutura de Pastas

Após extrair, você deve ter essa estrutura:

C:\flutter\ ├── bin\ │ ├── flutter.bat │ └── dart.exe ├── packages\ └── README.md

🛣️ 3. Configuração do PATH

Método 1: Interface Gráfica (Recomendado)

  1. Pressione Win + R e digite:
    sysdm.cpl
  2. Clique na aba "Avançado"
  3. Clique em "Variáveis de Ambiente"
  4. Na seção "Variáveis do sistema", encontre "Path"
  5. Clique em "Editar"
  6. Clique em "Novo"
  7. Digite exatamente:
    C:\flutter\bin
  8. Clique "OK" em todas as janelas

Método 2: PowerShell (Temporário)

Para adicionar temporariamente nesta sessão:

$env:PATH += ";C:\flutter\bin"

📱 5. Criação ou Restauração de Projeto

💡 Dica de Ouro: Dependendo do seu objetivo agora, você pode iniciar um aplicativo do absoluto zero ou restaurar um projeto que já existe. Escolha o cenário abaixo.

Cenário A: Criar um Projeto do Zero

Passo 1: Navegar para o Diretório de Trabalho

cd "C:\Users\Eduar\Desktop\agenda em flutter"

Passo 2: Criar o Projeto

Passo 3: Entrar no Projeto

cd agenda_desenvolvedor

Cenário B: Restaurar um Projeto Existente

Passo 1: Entrar na Pasta do Projeto

cd "C:\Caminho\Para\O\Seu\Projeto\devlog"

Passo 2: Baixar Dependências

Este comando lê o arquivo pubspec.yaml e instala todos os pacotes necessários.

flutter pub get

▶️ Passo Comum: Executar o Projeto

Para Web (Chrome):

flutter run -d chrome

Para Desktop (Windows):

flutter run -d windows

Para ver dispositivos disponíveis:

flutter devices

🔧 6. Tratamento de Erros Comuns

❌ Erro: "O termo 'flutter' não é reconhecido"

Problema: Flutter não está no PATH ou PATH não foi atualizado.
Soluções:
  1. Verificar se o PATH foi configurado corretamente:
    $env:PATH -split ";" | Where-Object { $_ -like "*flutter*" }
  2. Adicionar temporariamente:
    $env:PATH += ";C:\flutter\bin"
  3. Reiniciar o computador para aplicar PATH permanente
  4. Verificar se o arquivo existe:
    Test-Path "C:\flutter\bin\flutter.bat"

⚠️ Aviso: Gradle/Java Compatibility

Problema: Versão do Java incompatível com Gradle.
Soluções:
  1. Configurar JDK compatible:
    flutter config --jdk-dir=<JDK_DIRECTORY>
  2. Atualizar versão do Gradle no arquivo:
    android/gradle/wrapper/gradle-wrapper.properties

🔧 Problemas com Android SDK

Problema: cmdline-tools component missing ou licenças não aceitas.
Soluções:
  1. Aceitar licenças Android:
    flutter doctor --android-licenses
  2. Instalar cmdline-tools via Android Studio
  3. Para desenvolvimento web apenas, estes erros podem ser ignorados

🛠️ Erro ao construir Windows: gerador CMake errado

CMake falha com mensagem Generator Visual Studio 16 2019 could not find any instance of Visual Studio mesmo tendo apenas Visual Studio 2022/2026 instalado.
O que fizemos para resolver
  1. Identificamos o erro inicial: CMake usava o gerador "Visual Studio 16 2019" inexistente.
  2. Analisamos o código do Flutter em visual_studio.dart e vimos que o gerador era escolhido pelo número da versão principal; faltava tratamento para instalações muito novas.
  3. Aplicamos o ajuste no SDK (ou instalamos VS 2019/2022) e rodamos flutter clean para regenerar os arquivos de build; o erro de geração desapareceu.
  4. Durante a segunda tentativa apareceu conflito de pacote (intl), então atualizamos o pubspec.yaml para usar intl: ^0.20.2 e executamos flutter pub get.
  5. Por fim, flutter run -d windows compilou o executável e o app foi lançado com sucesso.
Diagnóstico e soluções adicionais:
  1. Verifique flutter doctor -v para confirmar a versão do Visual Studio detectada.
  2. Corretivos possíveis (válidos para qualquer projeto Flutter que sofra o mesmo erro):
    • Instalar o workload Desktop development with C++ ou uma versão suportada do VS (2019 ou 2022) ao lado de pré‑vias mais recentes.
    • Editar o SDK do Flutter para forçar o gerador certo. A modificação fica no arquivo C:\src\flutter\packages\flutter_tools\lib\src\windows\visual_studio.dart e afeta **todos** os projetos que usam esse SDK. O trecho alterado deve ficar assim: 
      String? get cmakeGenerator {
  // Fluxo de fallback para instalações muito novas/preview que não retornam
  // um número de versão principal válido. Sem isso, CMake escolheria o
  // gerador "Visual Studio 16 2019" por padrão.
  if (_majorVersion == null) {
    return 'Visual Studio 18 2026';
  }
  return switch (_majorVersion) {
    18 => 'Visual Studio 18 2026',
    17 => 'Visual Studio 17 2022',
    _  => 'Visual Studio 16 2019',
  };
}
      Após salvar, abra um novo terminal, execute flutter clean dentro de qualquer projeto afetado e rode flutter run -d windows novamente. O patch persistirá enquanto você usar esse SDK (se atualizar o Flutter, pode reaplicar ou aguardar correção upstream).
  3. Se surgir conflito de versão de pacotes, independente de plataforma, ajuste as dependências no pubspec.yaml do projeto (por exemplo, atualizar intl) e rode flutter pub get novamente.
⚠️ Este resumo agora fornece exemplos concretos e linguagem genérica que podem ser copiados para futuros projetos com o mesmo erro.
